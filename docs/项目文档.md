# 产品需求文档-PRD

## 一、基础信息
工业测控代码生成软件以AI赋能DaoSCADA平台，实现数据模型、组态界面、绑点信息、控制脚本等代码智能化生成，替代人工完成界面搭建、设备建模、绑点配置等耗时工作，提升组态效率，保障生成内容的规范性和可用性。


## 二、技术要求

### 2.1 功能要求
- 软件能够基于设备说明书等描述文件自动生成DaoSCADA平台设备模型；
- 软件能够基于开发者自然语言描述，结合DaoSCADA控件库设计准确、美观、实用的组态界面，并生成符合DaoSCADA组态界面描述标准的json文件；
- 软件能够自动生成设备模型与PLC的绑点文件；
- 软件能够生成DaoSCADA支持的控制脚本代码；
- 软件能够以插件等方式集成到DaoSCADA系统中。

### 2.2 架构要求
- 后端技术：采用FastAPI框架开发，支持高并发请求处理，接口响应时间≤500ms（不含第三方API调用耗时）；支持跨平台部署（WindowsServer2019及以上、CentOS8及以上）。
- 前端技术：采用Vue/React框架开发，支持Canvas/SVG渲染，预览页面支持缩放、拖拽操作，适配1920×1080及以上分辨率，无卡顿、无渲染异常。
- 数据存储：素材库采用SQLite+文件系统组合存储，向量库采用Chroma本地部署，支持离线运行；数据存储需符合工业数据安全规范，防止数据丢失、泄露。
- 第三方依赖：LLM API、文生图API需选用开源或经甲方确认的商业接口，接口调用稳定，故障率≤0.5%；所有第三方依赖需提供版本说明及兼容性证明。

### 2.3 开发规范
- 代码规范：遵循Python/JavaScript行业标准编码规范，代码注释率≥30%，提供详细代码说明文档；采用Git进行版本控制，完成一个功能模块需提交对应版本标签。
- 文档规范：所有技术文档（技术方案、规则说明、测试报告等）需采用统一格式，内容完整、逻辑清晰，术语统一，符合工业软件文档规范。
- 兼容性要求：生成的组态描述JSON文件需兼容DaoSCADA现有渲染引擎，可直接被解析渲染；系统支持与DaoSCADA平台现有功能模块无缝对接，无兼容性冲突。


## 三、关键流程

### 3.1 JSON Schema规则库及约束规则构建
- 规则规范：严格遵循JSON Schema Draft-07标准，定义组态描述文件三大顶层结构(canvas,control,binding)：画面、控件、绑定,枚举值范围。
- 枚举约束：所有枚举值需通过字典文件硬编码，禁止AI生成枚举外的值；枚举字典需单独成册，明确每个枚举值的含义及使用场景。
- 校验工具：开发Python校验CLI工具，支持对任意JSON文件进行Schema校验，能精准识别字段缺失、类型错误、枚举越界等问题，并输出具体错误行、错误类型及修改建议。
- 验收指标：校验工具对合法/非法JSON文件的判断准确率100%；提供至少3个合法示例JSON和3个非法JSON，均能被工具正确识别；提交完整的规则文件、数据枚举文件、校验工具及规则说明文档。

### 3.2 AI素材/控件生成Agent
- Query处理：支持自然语言Query输入（如“水泵运行状态指示灯”），能自动改写Query并提取关键实体（设备类型、状态、风格），提取准确率≥90%。
- 检索与生成：优先在本地素材库进行语义检索（基于embedding相似度），相似度＞0.85时直接复用素材；库中无匹配时，调用文生图API生成工业风格控件图标，生成素材符合工业组态界面风格，清晰度≥128×128像素，无模糊、变形。
- 素材管理：生成的素材经自动质检（尺寸、格式、风格）后入库，存储完整元数据；提供简单Web管理界面，支持素材浏览、搜索、删除操作，操作响应时间≤100ms。
- 批量生成：支持输入设备类型列表，批量产出对应控件素材，批量生成效率≥10个/分钟，生成素材合格率≥95%。
- 验收指标：5条不同Query检索准确率≥90%；新生成素材能成功入库并正常展示；素材库管理界面功能正常，无操作异常。

### 3.3 AI自动布局
- 前置处理：能准确读取画布宽高、素材列表（含尺寸）及用户Query，提取关键布局需求，提取准确率≥90%。
- 布局生成：先通过Mermaid/流程图DSL描述控制流骨架，再基于骨架规划区域分布，生成符合Schema的JSON，包含每个控件的x/y/width/height坐标，坐标精度≤1px。
- 布局约束：硬编码布局约束规则，功能相近控件聚合（同类设备距离＜200px）；管路控件必须与设备控件有连接关系；仪表/显示类控件默认置于右侧面板区；约束规则执行率100%。
- 兜底算法：元素数量＞20的大型场景，引入力导向图布局算法（D3-force或自实现）辅助排版，AI负责微调，确保布局合理、无明显拥挤。
- 预览与质检：后端生成JSON后，前端Web页面实时解析渲染，支持缩放、拖拽查看，渲染延迟≤1000ms；布局质检工具能检测控件重叠（重叠率＞10%报警）、孤立控件（无连接的非显示类控件报警），检测准确率100%。
- 验收指标：3个典型场景的布局JSON均通过Schema校验；预览页面正常渲染，无控件重叠、管路连接错误；布局逻辑经人工评审合格。

### 3.4 微调接口
- 元素选择：前端预览页面支持点击/框选单个或多个控件，选中后可右键调用AI助手，选择准确率100%，无误选、漏选。
- 上下文传递：选中元素后，自动提取其id、type、x、y等属性，连同画布完整JSON作为上下文发送给AI，属性提取完整率100%。
- 指令响应：支持自然语言调整指令（换位、改色、对齐、缩放、删除等），AI仅返回JSON Patch（RFC 6902格式）差异包，不返回全量JSON，响应时间≤1000ms；指令响应准确率≥80%。
- 预览与确认：前端应用JSON Patch渲染差异预览，提供“接受修改”“撤销”按钮，撤销功能可完全回滚至修改前状态，无数据残留；修改后自动更新内存中的完整JSON，确保后续操作基于最新状态。
- 验收指标：5条典型调整指令响应准确率≥80%；撤销功能完全回滚；修改历史可正常查看，无丢失。

### 3.5 数据绑点
- 清单解析：支持上传Excel格式设备变量清单，能模糊匹配列名，提取变量名、数据类型、寄存器地址等关键字段，解析准确率≥95%。
- 匹配算法：基于embedding语义相似度，自动关联组态控件与后台变量，输出候选匹配列表；控件label与变量名模糊匹配（如“水泵1”与“Pump_01”）时，能通过同义词/编号对应关系匹配，并在日志中说明匹配依据。
- 冲突检测：硬编码冲突检测规则，能准确识别同一寄存器地址多控件绑定（冲突）、变量名重复（冗余），输出详细冲突报告，检测准确率100%。
- 复核界面：提供Web表格复核界面，列出所有控件及其匹配变量，支持手动修改、确认，操作便捷，无卡顿。
- 验收指标：标准清单自动匹配准确率≥75%（人工评审Top-1准确率）；能正确识别3条预置冲突场景；生成的绑点JSON通过Schema校验。
